базы данных контрейт файл

приложение - бесконечный цикл, который работает постоянно

два типа баз данных:
- реляционные - вся информация структурирована - информация разложена как в таблице
- нереляционные - менее четкая структура информации - информация хранится в форматое json файлов

В ЛР будем юзать SQLite - реляционная.

Для работы с БД используют язык SQL.
4 основных операции:
create
read
update
delete

Данные хранятся в виде таблиц. У всех строк есть уникальный primary key, по которому к информации можно обращаться.
Можно сделать любую колонку уникальный.

У каждой строки может быть снешний ключ, который явзяется первичным ключем в другой таблице - так могут связываться несколько таблиц.
Связи могут быть: один ко многим, многие ко многим.

import sqlite3
sql_connect = sqlite3.connect(database)
sql_cursor = sql_connect.cursor()
все данные из таблиц приходит в формате таблиц
доставать в код всю таблицу нельзя, нужно каждый раз делать запрос в таблицу

контрейд файл - для записи каких-то мгновенных измерений
для просмотра осйиллограм используют waves - годится для просмотра контрейд файлов
контрейд состоит из двух файлов: cfg и dat
cfg - для настройки осциллограмм
dat - сами данные

задача ЛР - считать из контрейд файла и записать в бд + рассчитать действующие значения
